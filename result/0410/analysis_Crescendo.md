# Crescendo: The Multi-Turn LLM Jailbreak Attack

**来源**：USENIX Security 2025（Seattle，August 13–15）  
**作者**：Mark Russinovich, Ahmed Salem, Ronen Eldan（Microsoft）  
**文件**：Crescendo_Multi-Turn_Jailbreak_USENIX_Security_2025.pdf

---

### 1. Task（任务定义）

- **核心问题**：设计一种黑盒、多轮、无需对抗性文本的越狱攻击，使安全对齐的 LLM 生成其正常拒绝的有害内容。
- **形式化表达**：  
  - 输入：目标任务 T（如生成制毒步骤）、对目标模型 M 的黑盒 API 访问。  
  - 输出：使 M 最终输出满足 T 要求的有害内容的多轮对话序列 D = (q1, r1, q2, r2, ..., qn, rn)。  
  - 目标函数：最大化 Attack Success Rate (ASR)，约束为每轮 qi 对人类读者看起来良性无害、且无显式对抗性后缀或优化痕迹。

---

### 2. Challenge（传统挑战）

- **现有困境**：  
  - GCG 等基于梯度的对抗后缀方法需要**白盒访问**，对商业 API 无效，且计算成本高昂。  
  - DAN 类单轮文本越狱一旦被发现即可被**输入过滤器**拦截，因为提示本身含可识别恶意内容。  
  - PAIR、CoA 等 LLM 辅助越狱依然生成语义上含恶意指令的提示，**内容特征可被检测**。  
  - 现有多轮越狱（CIA、MSJ 等）要么在单次调用内预先构造所有轮次（CIA），要么依赖大量恶意上下文示例（MSJ），均在提示层留有**可检测恶意信号**。  
  - 所有主流基准与对齐训练均**只覆盖单轮越狱**，对多轮攻击的防御存在系统性盲区。

---

### 3. Insight & Novelty（洞察与创新）

#### 3.1 Inspiration（灵感来源）

- LLM 具有**跟随对话模式**的内在倾向，对近期上下文（尤其是自身上一轮输出）高度敏感。  
- 社会心理学中的**「登门槛」（foot-in-the-door）**效应：先让对方答应小请求，显著提升其后续答应更大请求的概率。  
- 利用模型**自身的输出**来引导下一轮提示，等价于让模型参与了「构建攻击弹药」的过程，且每轮提示在语义上独立看都是良性的。

#### 3.2 Insight（核心洞察）

- **策略层**：越狱不必一次完成——利用 LLM 的**序列一致性偏好**逐步累积语境，让模型在无感知的情况下「自己走进陷阱」。  
  - 对应灵感：LLM 对近期上下文和自身输出的高注意力权重。  
- **策略层**：以良性无害的问题开场，每轮基于模型**自身回复**递进提问，永远不在 qi 中出现明确的有害关键词。  
  - 对应灵感：登门槛效应 + 利用模型输出构建下一步语境。

#### 3.3 Novelty Type（创新类型）

策略层创新：提出新的越狱交互范式（多轮渐进式 + 无恶意提示），与现有技术在威胁模型和可检测性上形成质的区别。

#### 3.4 Novelty Details（创新详述）

- 1️⃣ 【解决的问题是】：单轮越狱的恶意提示内容可被过滤器识别。  
  -> 【受哪个 Insight 启发】：每轮提示保持良性，危险语境由模型自身输出逐轮积累。  
  -> 【具体创新设计】：Crescendo 每轮 qi 均为语义无害的问题（如「告诉我关于 X 的背景」→「那些持担忧立场的人会怎么说」→「把这些话整合成文章」），不使用优化后缀或明确恶意词汇，天然绕过基于输入内容的过滤机制。

- 2️⃣ 【解决的问题是】：人工逐步越狱效率低且难以规模化评估。  
  -> 【受哪个 Insight 启发】：LLM 可以充当攻击策略生成器，利用目标模型反馈动态调整。  
  -> 【具体创新设计】：Crescendomation 以目标任务 + API 为输入，用另一个 LLM 生成每轮 Crescendo 问题，内置 Judge（GPT-4 自评）+ Secondary Judge 双层成功判断 + 退出/回溯逻辑，实现全自动化越狱；在 AdvBench 子集上 ASR 比 PAIR 高 29–61%（GPT-4）、比 PAIR 高 49–71%（Gemini-Pro）。

- 3️⃣ 【解决的问题是】：越狱通常只针对文本模态。  
  -> 【受哪个 Insight 启发】：多模态系统的图像生成受同一 LLM 对话上下文控制。  
  -> 【具体创新设计】：文字越狱成功后追加图像生成指令，使系统生成其正常拒绝的图片，扩展 Crescendo 到多模态场景。

### 4. Potential Flaw（潜在缺陷与拓展）

#### 4.1 情境局限与延伸

- Crescendo 依赖**多轮累积**，单次调用后即清除上下文的无状态接口（如部分 API 配置）可打断攻击链。  
- 若系统引入**跨轮意图一致性检测**（如对话摘要+安全评分），Crescendo 的渐进式语境累积可能被识别出来。  
- 延伸：研究「多会话跨文件/工具」场景下的 Crescendo 变体——当 agent 调用工具并将工具输出拼接到下一轮时，攻击面扩展到工具链。

#### 4.2 数据脆弱性

- **任务依赖路径规划**：Crescendo 需要攻击者对任务有足够背景知识以设计合理的渐进路径，对高度专业化有害任务（如特定化工配方）的泛化能力受限。  
- **模型更新**：若目标模型进行了**多轮对话联合对齐训练**，针对单轮的对齐失效分析不再适用，ASR 会显著下降。  
- **评估工具本身的局限**：Judge 和 Secondary Judge 均为 GPT-4，存在同源模型评估偏差，且 GPT-4 本身可能拒绝宣布「攻击成功」（论文已指出这一 false negative 问题）。

#### 4.3 值得深挖的研究点

多轮跨工具 agent 场景下的 Crescendo 泛化：现有 Crescendo 仅研究纯对话 LLM。当 agent 具备工具调用能力（搜索、代码执行、文件读写）时，攻击者可以在工具输出中注入「下一轮渐进语境」，实现**工具辅助的 Crescendo**，绕过对对话历史的监控。这一交叉问题（多轮越狱 × agent 工具链）是当前文献的空白，可独立成一篇论文。

### 5. Motivation（思想逆向推演）

- [当前越狱都在想办法构造一条「越狱提示」让模型一次性回答...] -> [那本质痛点是什么？是「一次请求里的恶意信号太明显，被过滤器抓住了」...] -> [但人类在真实对话中影响他人观点，从来不是靠一句话，而是「一步步引导」——这个过程里每一句话都是无害的...] -> [LLM 的上下文窗口本质上是一个「记忆累积装置」，而且它倾向于延续自己上轮的输出逻辑...] -> [那能不能让模型在不知不觉中一步步「自己走到」有害输出，而每一步的提示对过滤器都是透明的？] -> [最终推导出：从无害起点出发，每轮基于模型自身回复递进，构造出 Crescendo 多轮越狱范式]
